BASELINE RAW SUMMARY QUERY

SELECT
    issuer AS [HIOS Issuer ID],
    insurance_type AS [Insurance Type],
    coverage_year AS [Coverage Year],

    COUNT(DISTINCT COALESCE(
        NULLIF(policy_id, ''),
        NULLIF(health_coverage_policy_no, '')
    )) AS [Our Enrollments Total],

    COUNT(DISTINCT COALESCE(
        NULLIF(member_id, ''),
        NULLIF(issuer_indiv_identifier, ''),
        NULLIF(exchg_assigned_enrollee_id, '')
    )) AS [Our Enrollees Total],

    COUNT_BIG(*) AS [Our Raw Rows]

FROM dbo.inbound_automation
WHERE folder_year IN (2025, 2026)
  AND coverage_year IN (2025, 2026)
  AND insurance_type IN ('Health', 'Dental')
GROUP BY
    issuer,
    insurance_type,
    coverage_year
ORDER BY
    issuer,
    insurance_type,
    coverage_year;

Meaning:
- Groups by issuer + insurance type + coverage year
- Counts distinct policies
- Counts distinct members
- Counts all raw transactions
- Does not apply canonical/business-ready rules

CRITICAL ANALYTICAL CAUTIONS

Never say:
- “The Best Life record was found” merely because the same enrollee ID exists under another issuer
- “The file is missing” merely because the member is absent from the Excel extract
- “The parser dropped the record” without proving the member exists in source XML and the loaded Azure file has fewer/different parsed rows
- “The coverage year is wrong” without showing which source date each system uses
- “The remaining 71 are all true missing records” without confirming the temporary-table source and deduplication

Preferred language:
- “Person-level match found under another issuer”
- “No match under issuer 83502 using the tested keys”
- “Currently unresolved”
- “Source-file investigation candidate”
- “This supports a hypothesis but does not prove it”

EVIDENCE REQUIRED TO CLASSIFY ROOT CAUSE

Parser drop:
Accept only if:
- XML contains the member
- The exact file was loaded
- Azure lacks that member from that file
- Parser/file row counts or raw payload prove omission

Extract/business filter:
Accept only if:
- Row exists under 83502 in Azure
- The Selma Excel query/filter excluded it

Issuer assignment difference:
Accept only if:
- Same person exists under another issuer
- No matching Best Life policy under 83502
- Workbench/Sisense shows 83502 for the target enrollment

Household mapping difference:
Accept only if:
- Policy/subscriber exists under 83502
- Dependent member is absent or represented differently in the same policy/source file

Missing source file:
Accept only if:
- Account History identifies a source file
- File is absent from SFTP/local archive and file log, or file log proves it never loaded

Loaded file/parser issue:
Accept only if:
- Account History source file exists in file log as loaded
- XML contains target member
- inbound_automation lacks target row for that file

1. Inspect available columns before assuming names

Status/benefit/date search:

SELECT
    ORDINAL_POSITION,
    COLUMN_NAME,
    DATA_TYPE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = 'dbo'
  AND TABLE_NAME = 'inbound_automation'
  AND (
        COLUMN_NAME LIKE '%benefit%'
     OR COLUMN_NAME LIKE '%effective%'
     OR COLUMN_NAME LIKE '%start%'
     OR COLUMN_NAME LIKE '%end%'

2. Search a member across all known enrollee identifiers

DECLARE @MemberId VARCHAR(100) = '1007765191';

SELECT
    issuer,
    insurance_type,
    coverage_year,
    folder_year,
    policy_id,
    health_coverage_policy_no,
    member_id,
    issuer_indiv_identifier,
    exchg_assigned_enrollee_id,
    enrollee_status,
    maintenance_type_code,
    member_maint_effective_date,
    source_file_name,
    row_number_in_file,
    file_hash,
    load_run_id,
    loaded_at
FROM dbo.inbound_automation
WHERE
       LTRIM(RTRIM(member_id)) = @MemberId
    OR LTRIM(RTRIM(issuer_indiv_identifier)) = @MemberId
    OR LTRIM(RTRIM(exchg_assigned_enrollee_id)) = @MemberId
ORDER BY
    issuer,
    coverage_year,
    member_maint_effective_date,
    loaded_at;

6. Batch missing-ID search using a temporary table

CREATE TABLE #SwathiMissingEnrolled
(
    enrollee_id VARCHAR(100) NOT NULL
);

-- Insert the verified Swathi IDs here.
-- Ensure the source is explicitly filtered to:
-- HIOS 83502, Dental, Coverage Year 2026, correct comparison category/status.

SELECT
    m.enrollee_id AS [Swathi Enrollee ID],
    CASE
        WHEN ia.member_id IS NOT NULL
          OR ia.issuer_indiv_identifier IS NOT NULL
          OR ia.exchg_assigned_enrollee_id IS NOT NULL
        THEN 'FOUND IN DB'
        ELSE 'NOT FOUND IN DB'
    END AS [DB Result],
    ia.issuer,
    ia.insurance_type,
    ia.coverage_year,
    ia.folder_year,
    ia.policy_id,
    ia.health_coverage_policy_no,
    ia.member_id,
    ia.issuer_indiv_identifier,
    ia.exchg_assigned_enrollee_id,
    ia.enrollee_status,
    ia.maintenance_type_code,
    ia.member_maint_effective_date,
    ia.source_file_name,
    ia.row_number_in_file,
    ia.file_hash,
    ia.loaded_at
FROM #SwathiMissingEnrolled AS m
LEFT JOIN dbo.inbound_automation AS ia
    ON  LTRIM(RTRIM(ia.member_id)) = LTRIM(RTRIM(m.enrollee_id))
    OR  LTRIM(RTRIM(ia.issuer_indiv_identifier)) = LTRIM(RTRIM(m.enrollee_id))
    OR  LTRIM(RTRIM(ia.exchg_assigned_enrollee_id)) = LTRIM(RTRIM(m.enrollee_id))

    [DB Result],

    ia.coverage_year,
    ia.loaded_at;
